# latte-chain

latte-chain 是一个使用 Rust 实现的区块链学习项目，目标是**系统性实现一个功能完整的区块链系统**，并在实现过程中明确各核心模块的职责边界与工程抽象。

本项目面向学习与研究，强调设计清晰度、模块拆分合理性以及演进过程的可理解性，而非直接追求生产级性能或安全性。

---

## 项目目标

* 从零实现一个结构完整的区块链系统
* 覆盖区块链的核心子系统，而非局部 Demo
* 通过 Rust 的类型系统和模块系统，强化对系统边界的理解
* 为后续深入研究共识、状态机复制等问题打下工程基础

---

## 功能范围

### 已规划 / 目标内功能

* 区块与区块头结构（Block / Header）
* 交易模型与交易池（Transaction Pool）
* Merkle Tree 与数据完整性校验
* 区块校验（Block Validation）
* 区块执行与状态转换（Block Executor）
* 状态存储与状态校验（State / State Validation）
* 共识算法（初期为简化共识，后续可演进）
* P2P 网络与区块、交易传播
* 单节点到多节点的完整出块与同步流程

### 明确不作为当前目标的内容

* 极致性能优化
* 完整经济模型与复杂激励机制
* 生产环境安全加固

## 设计原则

* 模块职责单一且边界清晰
* 校验（Validation）、执行（Execution）、共识（Consensus）相互解耦
* 校验逻辑尽量保持确定性与无状态
* 状态变更只能通过执行器产生
* 抽象按需引入，避免过早设计

---

## 关于区块校验与状态校验

当前阶段实现的区块校验主要关注：

* 区块结构合法性
* Hash 与 Merkle Root 的正确性
* 区块之间的前后依赖关系

后续将逐步引入：

* 基于状态的校验规则
* 执行结果与状态根校验
* 共识相关约束条件

## 演进路线（高层）

1. 完成交易模型与交易池
2. 实现最小可用的区块执行与状态存储
3. 引入简化共识流程，跑通单节点闭环
4. 加入 P2P 网络，实现多节点同步
5. 强化状态校验与共识规则
6. 回顾并重构整体模块边界

## 项目定位说明

本项目是一个**以工程实现为核心的区块链学习项目**，适合希望通过代码深入理解以下问题的开发者：

* 区块链系统为何需要这样的模块划分
* 共识、执行与状态之间的关系
* 一个区块从产生到被网络接受的完整生命周期
